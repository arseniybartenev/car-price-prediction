## Описание проекта

Сервис по продаже автомобилей с пробегом «Не бит, не крашен» разрабатывает приложение для привлечения новых клиентов, позволяющее узнать рыночную стоимость своего автомобиля. Для этого необходима модель машинного обучения, способная точно определять цену на основе технических характеристик, комплектации и данных о других автомобилях.

**Цель:** Построить модель регрессии для предсказания рыночной стоимости автомобиля. Критически важными для заказчика являются качество предсказания, а также скорость обучения и инференса модели.

## Данные

В распоряжении имеется датасет `autos.csv` с информацией об автомобилях, содержащий следующие признаки:

- **Признаки:** дата скачивания анкеты, тип кузова, год регистрации, тип коробки передач, мощность (л.с.), модель, пробег (км), месяц регистрации, тип топлива, марка, наличие ремонта, дата создания анкеты, количество фотографий, почтовый индекс, дата последней активности пользователя.
- **Целевой признак:** `Price` — цена автомобиля в евро.

## Этапы работы

1. **Загрузка и первичный анализ данных**
   - Загрузка данных, изучение общей информации, выявление пропусков и аномалий.

2. **Предобработка данных**
   - Удаление неинформативных признаков (даты, количество фото, почтовый индекс и др.).
   - Обработка аномальных значений в признаках `Price`, `RegistrationYear`, `Power`, `RegistrationMonth`.
   - Обработка пропусков в категориальных признаках (заполнение модой по бренду, либо значением 'unknown').
   - Переименование колонок для удобства.

3. **Исследовательский анализ данных (EDA)**
   - Визуализация распределений количественных и категориальных признаков.
   - Выявление аномалий и особенностей данных (например, ступенчатое распределение пробега).
   - Анализ топ-моделей для популярных брендов.

4. **Подготовка признаков к обучению**
   - Кодирование категориальных признаков (One-Hot Encoding, Label Encoding).
   - Масштабирование числовых признаков (StandardScaler).
   - Разделение данных на обучающую и тестовую выборки.

5. **Обучение моделей и анализ времени работы**
   - Обучение моделей: линейная регрессия (базовая), дерево решений, CatBoost, LightGBM.
   - Измерение времени обучения и времени предсказания для каждой модели.

6. **Оценка качества на тестовой выборке**
   - Выбор лучшей модели по метрике RMSE и с учётом скорости работы.
   - Рекомендации по внедрению в приложение.

## Используемые технологии

- **Python** — основной язык.
- **Pandas, NumPy** — обработка и анализ данных.
- **Matplotlib, Seaborn** — визуализация.
- **Scikit‑learn** — предобработка, пайплайны, линейная регрессия, дерево решений, метрики.
- **CatBoost, LightGBM** — градиентный бустинг для сравнения производительности.
- **phik** — расчёт корреляций для смешанных типов данных.

## Результаты

| Модель               | RMSE (тест) | Время обучения | Время предсказания |
|----------------------|-------------|----------------|---------------------|
| LinearRegression     | 2541.92     | 0.24 сек       | 0.004 сек           |
| DecisionTreeRegressor| 2098.43     | 0.69 сек       | 0.009 сек           |
| CatBoostRegressor    | **1696.97**  | 2.73 сек       | 0.044 сек           |
| LGBMRegressor        | 1781.48     | 1.25 сек       | 0.203 сек           |

Лучшей моделью по сочетанию качества и скорости показал себя **CatBoost**, достигнув RMSE = 1696.97.

## Выводы

Разработанная модель может быть интегрирована в приложение сервиса «Не бит, не крашен» для автоматического определения рыночной стоимости автомобиля. Ожидаемый эффект:

- Предоставление пользователям точной и быстрой оценки стоимости их автомобиля.
- Повышение доверия к сервису и привлечение новых клиентов за счёт удобного и полезного инструмента.

**Ключевые факторы успеха:**
- Тщательная очистка и предобработка данных (обработка аномалий и пропусков).
- Использование градиентного бустинга (CatBoost), хорошо работающего с разнотипными признаками.
- Сравнение не только по качеству, но и по времени обучения/инференса, что критично для реального приложения.

## Запуск проекта

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/arseniybartenev/car_price_prediction.git
   ```
2. Перейдите в папку проекта:
   ```bash
   cd car_price_prediction
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Поместите файл `autos.csv` в папку `data/` (если папки нет, создайте её). В ноутбуке путь к данным можно настроить соответствующим образом.
5. Запустите Jupyter Notebook:
   ```bash
   jupyter notebook notebooks/determining_the_cost_of_cars.ipynb
   ```
